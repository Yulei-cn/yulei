RESUME

1. On the whole, the overall style of the website is relatively uniform, but the design of the page structure is relatively simple. There are still some deficiencies in the grasp of the more complex page structure. When there are more attributes such as floating, it is easy to get confused.
2. Since the production process is carried out simultaneously with the teacher’s class, the compatibility issues of different browsers and the page zoom problem after device changes were not considered at the beginning of the design. The homepage page produced at the beginning has some layout layout after the browser is reduced. There will be some changes. Post-production web pages have relatively few of these problems.
3. During the production process, I found that VS code can be used to write code, which is a little easier to use than DW.
4. In the production process of the website, it is recommended to build a good structure first, and then add content to it. If you think about it a little bit, it is easy to have problems later, and it is more difficult to change the structure.
5. Back up frequently during production to avoid wasted work due to failure to save in time. You can create a GITHUB copy to test the effect, so that it will not affect the production of the original. You can use more comments to debug the effect. If the effect is not good, you can directly remove the comment without affecting the original effect.
6. Learning materials can not only be textbooks, but also can be collected with the help of the Internet. Some forums and social media are good learning platforms.
